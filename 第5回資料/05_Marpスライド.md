## 【タスク1】自動化アイテムの実装

### 仕組み：2つの新しい要素

#### 1. アイテム購入ロジック

```javascript
let autoCPS = 0;  // 1秒あたりの自動増加量
const itemCost = 10;

upgradeBtn.addEventListener("click", () => {
    if (coin >= itemCost) {      // コインが足りてるか？
        coin -= itemCost;        // コインを消費
        autoCPS += 1;            // 自動パワーを+1
        localStorage.setItem("savedAutoCPS", autoCPS);
    }
});
```

#### 2. タイマー処理

```javascript
setInterval(() => {
    if (autoCPS > 0) {
        coin += autoCPS;         // 毎秒、自動増加
        coinCountEl.textContent = coin;
        localStorage.setItem("savedCoin", coin);
    }
}, 1000);  // 1000ミリ秒 = 1秒
```

<!-- 
話す内容:
- `if (coin >= itemCost)` という条件分岐で、買えるかチェック
- `setInterval(..., 1000)` で「1秒ごとに」という指示をPC に与えます
-->

## 🔄 関数化のススメ（発展課題①）

### クリック時と自動増加で同じ処理をしていませんか？

```javascript
// ❌ 同じコードが2か所に...
coin = coin + 1;
coinCountEl.textContent = coin;
// ✅ 関数化してスッキリ
function plusCoin(amount) {
    coin += amount;
    coinCountEl.textContent = coin;
    localStorage.setItem("savedCoin", coin);
}
// どこからでも呼べる
clickBtn.addEventListener("click", () => plusCoin(1));

setInterval(() => {
    if (autoCPS > 0) plusCoin(autoCPS);
}, 1000);
```

<!-- 
話す内容:
- 「DRY の原則」= Don't Repeat Yourself
- 同じコードが複数箇所にあるのは保守性が悪い
-->

## 【タスク3】世界への公開

### git pushの呪文

```bash
# ローカルの変更をステージング
git add .

# 変更をコミット（説明文付き）
git commit -m "オリジナルクリッカー完成！"

# リモートにプッシュ（世界に公開）
git push origin main
```

### ✅ 確認ステップ
