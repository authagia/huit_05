## **第4回：機能の分解とコアロジックの実装**
* **テーマ**: リロードしても消えない「コインクリッカー」のコアを作ろう
* **題材**: コインクリッカー（あえて平凡なモチーフにし、新入生のアイデアを邪魔しない）
* **目標**: 全員が「クリックすると数字が増え、ページを更新してもデータが残る」状態にする。

### 🕒 タイムライン（計2.5時間）
* **【15分】前回の振り返り＆今日のゴールデモ**
  * 講師が作ったシンプルな「コインクリッカー」を動かして見せる。
* **【20分】設計ワーク：機能を日本語に「分解」してみる**
  * コードを書く前に、スライドを使って「ボタンを押したら数字が増えて保存される」という仕組みを日本語のステップに分解する。
  * *分解例*: ①ボタンが押されたことを検知する ➔ ②変数に1足す ➔ ③画面の数字を書き換える ➔ ④LocalStorageに保存する
* **【40分】実装タイム①：クリックでカウントアップ（計時：20分×2）**
  * 配布したテンプレート（HTMLの骨組みと、空の関数のガワ）をもとに実装。
  * 20分ごとに進捗を区切り、遅れている人を既存メンバーがサポート。
* **【10分】休憩**
* **【40分】実装タイム②：データの保存（LocalStorage）（計時：20分×2）**
  * ブラウザを閉じてもコインが消えない魔法（LocalStorage）を実装。
* **【25分】ピアチェック（隣の人と確認）＆次回予告**
  * 「動かなくてもOK」という前提で、隣の人に現在の画面を見せ合う。
  * 既存メンバーは「データの保存できた？」と声をかけ、動いた人を褒めて活気を作る。

## **第5回：アイテム追加と「自分らしさ」の爆発（拡張とUI）**
* **テーマ**: 世界に一つだけのオリジナルクリッカーを完成させて遊ぼう
* **目標**: コインを消費するアップグレード要素を追加し、見た目を自由にカスタマイズして公開する。

### 🕒 タイムライン（計2.5時間）
* **【15分】今日の流れ説明＆「拡張」のロジック解説**
  * 「コインを10枚消費して、1クリックあたりの増加量を+1する」というロジックを解説。
* **【40分】実装タイム①：自動化・強化アイテムの追加（計時：20分×2）**
  * 条件分岐（`if`）を使って、コインが足りる時だけアイテムが買えるロジックを組む。
  * **★得意な人向け「エクストラクエスト」の提示**：
    * *クエスト1*: アイテムを買うたびに価格がインフレする数式を入れてみよう。
    * *クエスト2*: タイマー処理（`setInterval`）を使って、放置しても自動で増えるようにしよう。
* **【30分】デザインカスタマイズ（計時：15分×2）**
  * コインの画像を自分の好きな画像（フリー素材や自分の好きなキャラクターなど）に差し替える。
  * 背景色やフォントをCSSで自由に変えて「自分だけのゲーム」にする。
  * 完了したら `git push` して GitHub Pages を更新。
* **【10分】休憩**
* **【45分】Discord大交流会＆テストプレイ会**
  * 全員が自分のURLをDiscordのチャンネルに貼り付ける。
  * お互いのページにアクセスして遊び、「単位クリッカーになってるじゃん！」「自動化のインフレ率エグい」など、ワイワイ歓談する。
  * 最後に「来月のミニハッカソンでは、こういう仕組みを使って自由にアプリを作るよ！」と繋げる。
* **【10分】片付け・アンケート**

## **運営・ファシリテーションのポイント**

### 1. 進度差のコントロール（置いていかれる人の許容と救済）
* 解説スライドの直後に「回答例コード」をスライドに映す、またはDiscordに文字として投稿します。
* つまずいて置いていかれそうな人には、既存メンバーが「このコードをコピペして、まずは動く楽しさを味わおう！」と声をかけ、その場で完全に立ち止まらせないようにします（後からのサポートで全体最適化）。

### 2. メリハリのためのタイマー運用
* 各作業時間は必ずプロジェクター等にタイマーを映して計時します。
* 「あと3分です。終わっていない人は一度手を止めてスライドを見てください」とアナウンスし、ダラダラと作業が伸びて説明を聞き逃すのを防ぎます。

### 3. オリジナリティの余白
* ロジック（JS）と見た目（HTML/CSS）を切り離したテンプレートにすることで、プログラムが苦手な子でも「画像と文字を変えるだけ」で100%オリジナリティを出せるように設計しておきます。
