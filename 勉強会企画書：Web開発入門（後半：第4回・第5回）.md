## **🏁 第4回・第5回の連続ストーリー**

* **第4回**: 個人のブラウザ内で「ゲームの核（カウント・保存）」を完成させる。その際、**「ランキングに載せる名前（playerName）」や「総クリック数（clickCount）」といった伏線データをあらかじめフロント側に仕込んでおく。**  
* **第5回**: 講師陣が用意した「Hono \+ Cloudflare Workers」の爆速スターターキットを使い、各自のAPIサーバーをクラウドにデプロイ。個人のセーブデータをサーバーへ送信（POST）し、全員のデータを集計した「リアルタイム・サークル内スコアボード（GET）」をフロントに描画して大交流会を開く。

## **📊 【第4回：フロントエンド統合版】要素とタスク一覧**

第4回は、前回までに議論した「関数化」や「アンロック機能」などの自走要素をすべてこの回に集約し、さらに第5回へのデータ的な仕込みを行います。

### **⚙️ 習得する技術・ロジック**

* **データ永続化**: localStorage（setItem / getItem）  
* **データ構造の拡張**: playerName や clickCount をオブジェクトや変数として定義する。  
* **リファクタリング**: クリック時と自動増加時の処理を共通化する「関数化（plusCoin()）」。  
* **動的UI制御**: style.display を使った、特定枚数での「上位アイテムボタンのアンロック（出現）」。

### **📋 スプシ進捗ボードのタスク**

1. **準備完了**: テンプレートを開き、画面が出たら ◯  
2. **クリック＆関数化**: ボタンを押すと数字が増え、ロジックを関数にまとめられたら ◯  
3. **セーブ完了**: リロードしてもデータが残り、名前（playerName）の入力欄を作れたら ◯  
4. **クエスト挑戦**: 100枚でボタンが出現する「アンロック機能」などの発展課題に進んだら ◯

## **🌐 【第5回：バックエンド・ランキング編】要素とタスク一覧**

第5回は「通信」と「データの集約」がテーマです。不正対策はおもてに出さず、「自分のデータが世界に送られ、みんなの様子がスコアボードで見える感動」を全面に押し出します。

### **⚙️ 習得する技術・ロジック**

* **環境構築（最速体験）**: npm run deploy による Cloudflare Workers への一発デプロイ。  
* **非同期通信**: フロントエンドから fetch(URL, { method: "POST" }) でサーバーへデータをセーブする処理。  
* **APIの理解**: POST /update（データ送信）と GET /ranking（ランキング取得）の役割。  
* **動的描画（Webアプリの王道）**: サーバーから返ってきたJSON（ランキングデータ）を、JSのループ処理でHTMLのリスト（\<ul\> や \<table\>）に組み立てて表示する。

### **📋 スプシ進捗ボードのタスク**

1. **クラウド起動**: 配布されたHonoテンプレートをデプロイし、自分専用のサーバーURLを取得できたら ◯  
2. **データ送信成功**: ゲーム内で「セーブ」を押した時、自分のスコアがサーバーに送られるようになったら ◯  
3. **スコアボード出現**: ランキングデータをサーバーから読み込み、画面にみんなの順位が表示されたら ◯  
4. **大交流会＆提出**: Discordに自分のゲームのURLを貼り、みんなのゲームを遊び倒したら ◯

## **🛠️ 講師・運営陣が事前に準備（仕込み）すること**

第5回を「環境構築エラー」で破綻させないために、運営側で以下のテンプレート（スターターキット）を完全に作り込んでおく必要があります。

1. **バックエンドのテンプレート（Hono \+ Workers）**  
   * 新入生が触るのは src/index.ts のみ。  
   * POST /update で送られてきたデータ（名前・コイン数など）を Cloudflare KV に保存するロジックを**ほぼ完成状態で記述**しておく（新入生には1〜2行の簡単な書き換えや、称号システム用の条件分岐だけを弄らせる）。  
   * GET /ranking でKVから全プレイヤーのデータを引っ張ってきて、スコア順にソートしたJSONを返すロジックをあらかじめ実装しておく。  
2. **フロントエンド側の通信パーツ**  
   * fetch("自分のサーバーURL/update", { ... }) の具体的な書き方のスニペット（コピペ用）を用意しておく。
