# ワークシート

## 1. ハンズオン課題
1. `playerName` の入力欄を作り、表示内容に反映させよう
2. `clickCount` を `localStorage` に保存して、リロード時に復元しよう
3. 100枚に達したら、新しいボタンを表示させよう
4. `saveButton` を押すと、名前とコイン数が保存されるようにしよう
5. リロードすると一度データが消える体験を確認しよう
6. `10_テンプレート.html` のコメントを読んで、`11_サンプルコード.js` の穴埋めをしてみよう

## 2. 確認チェックリスト
- [ ] `playerName` が入力できる
- [ ] クリックでコイン数が増える
- [ ] `saveButton` を押すと `localStorage` に保存される
- [ ] ページをリロードしてもデータが復元される
- [ ] 100枚でアンロックエリアが表示される
- [ ] 自分のクリッカーゲームのテーマを考えてみた
- [ ] 友だちのゲームを1つ以上触ってみた

## 3. 発展課題
- `clickCount` が 300 以上のときに別のメッセージを出す
- `playerName` が未入力のときは保存できないようにする
- セーブした回数を画面に表示する
- 150枚で「自動化アイテム」ボタンを追加する
- 200枚でアニメーション付きのアンロックボタンを表示する

## 4. テストプレイ大会用チェック
- 自分のゲームを誰かに見せて、操作してもらう
- 他の人のゲームを遊んで、面白かった点をメモする
- 自分のゲームに「テーマ変更」や「ボタンの見た目」の改善アイデアを追加する

## 5. 動作確認のコツ
- ブラウザのデベロッパーツールで `localStorage` を確認する
- 正しいキー名が保存されているかを見る
- リロード前後のコイン数を比べる
- エラーが出たら、コンソールのメッセージを確認する
