## 💡 【自走への伏線①】「手動クリック」と「自動化」の共通点

今書いたコードをよく見てみましょう。

```javascript
coin = coin + 1;
coinCountEl.textContent = coin;

```

ここでやっている「変数を増やして画面を書き換える」という処理は、次回の第5回で実装する「放置していても勝手に増える自動化」でも**全く同じ処理**を使います。

* **手動クリック：** 「人間がボタンを押した瞬間」に、変数を増やして画面を書き換える
* **自動で増加：** 「タイマーが1秒刻んだ瞬間」に、変数を増やして画面を書き換える

「きっかけ（トリガー）」が変わるだけです。これらを一つのカタマリ（関数）にまとめておくと、後から自動化や強化アイテムをいくらでも追加できるようになります。これがプログラミングの「設計」の魔法です。

## 💾 【タスク3】セーブ完了：LocalStorage（作業時間：20分）

今の状態だと、ページをリロード（再読み込み）すると0枚に戻ってしまいます。ブラウザを閉じてもデータを記憶してくれる「LocalStorage（ローカルストレージ）」という金庫を使います。

* **データを金庫に入れる（保存）**：`localStorage.setItem("キーの名前", 値);`
* **データを金庫から出す（読み込み）**：`localStorage.getItem("キーの名前");`

### 🛠️ 実装手順

#### 1. クリックした時に「金庫へ保存」する

```javascript
clickBtn.addEventListener("click", () => {
    coin = coin + 1;
    coinCountEl.textContent = coin;

    // 【追加】"savedCoin" という名前で金庫に保存
    localStorage.setItem("savedCoin", coin);
});

```

#### 2. ページを開いた時に「金庫から読み込む」

テンプレートの `let coin = 0;` の部分を、以下のように書き換えます。

```javascript
// 修正後：金庫からデータを取る。もし空っぽなら「0」にする
let coin = Number(localStorage.getItem("savedCoin")) || 0;

// 起動した瞬間に、読み込んだコインの数を画面に表示する
coinCountEl.textContent = coin;

```

> 💡 `Number(...)` は、金庫から取り出した「文字」を「計算に使える数字」に変換するための魔法です。

### 🔍 動かしてみよう！

コインを増やした後、ブラウザをリロードしてみてください。**数字が消えずに残っていれば完全大成功です！** スプシの「3. セーブ完了」に `◯` をつけましょう！

## 🔑 【自走への伏線②】隠されたボタンの「アンロック」機能

ゲームが進むと、「最初は隠されていた強力なボタンが、コインが貯まると出現する」という仕組みが欲しくなりますよね。これも条件分岐（`if`）とCSSを組み合わせるだけで実装できます。

* **CSSの仕込み：** 最初はアイテムボタンを非表示（`display: none;`）にしておく
* **JSのロジック：** `if (coin >= 100)` になったら、ボタンを表示（`style.display = "block";`）に変える

データ（数字）の変化に合わせて、画面の見た目（UI）そのものを出現させる。これができるようになると、作れるアプリの幅が無限に広がります！早く終わった人は、今のうちからボタンを隠す方法を調べてみてください！
